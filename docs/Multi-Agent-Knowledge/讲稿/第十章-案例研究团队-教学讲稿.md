# 第十章《案例：研究团队》教学讲稿

## 讲授目标

本章根据 AutoGen 官方 Literature Review 示例，讲清 FunctionTool、AssistantAgent、RoundRobinGroupChat 和终止条件怎样组成研究团队。建议讲授 60～75 分钟。

## 一、开场：三个角色围绕一份综述协作

【页面提示：展示 Literature Review 执行流。】

用户提出综述主题后，RoundRobinGroupChat 按 Google Search Agent、Arxiv Search Agent、Report Agent 的顺序轮转。前两者分别搜索网页和论文，Report Agent 整合群聊中的证据写报告；出现 TERMINATE 后停止。

职责边界很清楚：搜索角色只提供信息，报告角色负责综合，避免多个 Agent 同时抢写最终报告。

## 二、AutoGen 的三层抽象

官方导入中的 `AssistantAgent` 表示可接收消息、调用模型和工具的角色；`FunctionTool` 把搜索函数包装成模型工具；`RoundRobinGroupChat` 表示团队调度；`TextMentionTermination` 检查终止文本。

三个角色通常共享 Agent 类，但通过 name、description、system message 和工具列表形成差异。这再次说明，角色差异不来自类名数量，而来自职责、提示和能力边界。

## 三、工具层：普通函数变成 FunctionTool

示例中的 `google_search()` 返回网页标题、链接和摘要，`arxiv_search()` 返回论文标题、作者、摘要和链接。`FunctionTool` 保存函数及自然语言描述，让模型知道工具参数和适用场景。

函数签名决定参数 Schema，描述则应明确区分网页检索与学术检索。如果描述含糊，模型可能在需要论文证据时调用普通网页搜索。运行前还要配置搜索服务和模型所需凭据。

## 四、Agent 层：按信息源分配工具

Google Search Agent 只绑定 Google 工具，系统提示要求它负责搜索而不写最终报告；Arxiv Search Agent 只绑定 arXiv 工具，输出论文标题、作者、摘要和链接。

这种设计没有造一个拿所有工具的“超级研究员”，而是按信息源拆分角色。好处是来源类型清楚、权限更小、错误更容易定位；代价是需要额外协调和去重。

Report Agent 不直接搜索，也没有把两个搜索 Agent 包成 `AgentTool`。它在自己的轮次读取群聊消息历史，从 Google Agent 的网页结果和 arXiv Agent 的论文结果中组织结构化综述，并在结尾输出 `TERMINATE`。

【页面提示：展示“角色、工具与消息边界”图。】

读图时要求学生区分两条关系：搜索 Agent 与 `FunctionTool` 是直接工具调用关系；搜索 Agent 与 Report Agent 之间则通过团队消息线程交接证据。不要把这两条关系讲成 Report Agent 嵌套调用搜索 Agent。

## 五、Team 层与停止条件

`RoundRobinGroupChat` 按参与者顺序轮流发言。`TextMentionTermination("TERMINATE")` 在消息出现指定文本时停止。这种机制简单、可解释，适合教学。

只依赖终止词并不充分。如果角色忘记输出 TERMINATE，团队可能不断搜索和改写。因此还应组合最大轮数、token、费用、无进展检测和人工取消。

## 六、任务怎样运行

官方示例用异步 `run_stream()` 接收“为构建多智能体系统的无代码工具写文献综述”这一任务，`Console` 持续展示模型消息、工具调用、工具结果和最终报告，`await` 等待团队结束。

流式事件使内部过程可观察：我们可以判断哪个搜索源返回了什么、Report Agent 是否正确引用、团队为什么继续或停止。最终答案之外的过程正是调试多 Agent 的证据。

## 七、怎样迁移成自己的研究团队

这个案例覆盖工具、角色、团队和停止四层。迁移时可以把 Google 换成企业知识库，把 arXiv 换成 PubMed、Semantic Scholar 或专利检索，保留 Report Agent 和显式停止条件。

迁移不是只改工具名。还要重新定义来源格式、去重、引用 Schema、失败处理和报告验收。如果用于正式研究流程，还应增加 FactChecker 或 Reviewer，验证每个 claim 能否回到来源。

## 八、练习与自检

练习时从搜索函数、FunctionTool、AssistantAgent 和 RoundRobinGroupChat 的调用关系开始，再替换主题或信息源，并记录工具调用与停止行为变化。新增数据结构或角色时，要说明它改变了哪一层职责。

请学生说出三个 Agent 的职责和工具边界，解释 FunctionTool、AssistantAgent、RoundRobinGroupChat 与 TextMentionTermination 的层次，并列出 TERMINATE 之外的三个护栏。

本章结论：研究团队的价值来自信息源分工、工具封装、证据综合和可控终止。下一章把两个案例放回多智能体框架生态，比较不同项目怎样封装角色、任务、消息和状态。
