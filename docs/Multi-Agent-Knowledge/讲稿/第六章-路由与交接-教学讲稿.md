# 第六章《路由与交接》教学讲稿

## 讲授目标

本章讲清分诊、规则优先路由、交接包、交接授权、失败升级和路由回归测试。建议讲授 55～70 分钟。

## 一、开场：选对人只是第一步

路由回答“谁接手”，交接回答“接收者能不能继续”。系统即使选中了正确 Agent，如果只发送“你继续处理”，接收者仍会缺少目标、产物、约束和未决问题。

【页面提示：展示路由决策树。】

收到任务后，依次识别任务类型、所需工具和风险，检查是否需要人工审批，再过滤有能力且有权限的角色。明确规则命中时直接路由，模糊情况才调用 Triage Agent。

交接状态机则包括 Received、Accepted、NeedClarification 和 Rejected。信息足够且有权限才接受；缺上下文要澄清；不属于能力范围或无权限要拒绝，而不是勉强执行。

## 二、Triage Agent

`RouteDecision` 要求分诊模型返回目标角色、理由、所需上下文和风险。Prompt 中应列出角色能力、允许目标和高风险转人工规则，输出还要经过白名单和风险策略。

Triage 不是万能主管。它适合语义模糊的分类，不应覆盖可以写成确定性规则的安全边界。

## 三、规则优先、模型回退

`deterministic_route()` 先把文本转小写；删除生产数据或发送外部邮件直接转人工；安全关键词转 SecurityReviewer；测试关键词转 Tester；没有命中才交给模型。

这种组合便宜、可预测、易回归。规则需要规范化、多语言覆盖和冲突优先级；模型结果仍需验证，不能路由到不存在或未授权角色。

## 四、Handoff Packet

`HandoffPacket` 明确发送者、接收者、任务、目标、摘要、产物、约束、未决问题和期望输出。它让接收者不必重读全部历史，也不会遗漏关键安全条件。

OAuth 示例由架构师交给开发者，目标特别强调不记录 token，设计文档通过稳定引用传递，期望输出包含补丁和测试。空的 open_questions 表示当前没有已知未决项，不代表忽略疑问。

## 五、交接函数与权限

`handoff()` 先检查目标 Agent 是否存在，再把 Pydantic 数据转成字典交给统一 `run()` 接口。检查必须发生在字典索引前，才能返回清楚错误。

`secure_handoff()` 进一步读取“发送角色可以交给哪些接收角色”的策略。低权限角色不能绕过 Supervisor，直接把任务交给部署者或人工审批角色。交接关系本身也是权限边界。

## 六、失败升级

`handle_agent_failure()` 按错误类型决定下一步：权限错误立即转人工；普通失败在有限次数内重试；超过次数升级 Supervisor；Supervisor 仍无法处理则转人工。

盲目重试没有价值。状态还应记录已尝试策略、错误分类和退避时间，避免同一失败重复消耗预算。升级包要携带失败证据，而不是只说“没成功”。

## 七、路由评测集

`ROUTE_CASES` 用代表性任务检查研究、测试、安全和生产部署是否进入预期目标。生产集还要覆盖模糊表达、冲突关键词、多语言、恶意提示、未知角色和模型回退路径。

路由指标包括准确率、人工升级率、误放行高风险任务率、平均成本和交接后澄清率。后者很重要：路由目标正确但交接信息不足，整体仍然失败。

## 补充讲授：把路由架构与真实项目逐项对照

【页面提示：展示“架构使用附录”表格。】

先讲直接证据：Swarm 和 OpenAI Agents SDK 都有 Triage 与 handoff；Magentic-One 的论文和 AutoGen 实现都展示了主管跟踪进度、在停滞后重规划；MasRouter 则提供了多智能体路由的论文、代码和效果—成本评测。

再讲部分证据与空缺。Agents SDK 的 typed handoff 和 input filter 能承载摘要、优先级并过滤历史，但不等于采用了网页中完整的 Handoff Packet。`is_enabled` 提供动态开关，也不等于 sender→allowed_targets 权限矩阵。规则优先、模型回退的精确实现，以及完整四信号路由树，都没有找到可核验的公开项目，因此表中明确写“无”。

最后让学生用一句话复述证据纪律：相关研究只能说明方向相近，字段、控制流和失败语义都对应时，才能称为同一架构。

## 八、章末复习与收尾

请学生区分路由与交接，列出 Handoff Packet 的最小字段，解释规则为什么优先于模型，以及权限错误为什么不应普通重试。

本章结论：可靠路由要选对接收者、满足权限并保留风险；可靠交接要携带足够但不过量的上下文。Blackboard 的共享状态机制在协作拓扑专题中展开，下一章讨论怎样观测和评测这些跨角色路径。
