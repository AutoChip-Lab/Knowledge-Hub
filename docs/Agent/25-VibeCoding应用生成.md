## 0. 全景:从「帮你写代码」到「替你造一个能上线的 app」

前面拆的那一长串——Claude Code、Codex、Cursor、Aider、自主 SWE agent——不管形态多不同,读者身份是**同一个**:**你是个会写代码的人**,agent 在你仓库里帮你读文件、改代码、跑测试、提 PR,交付**一个 diff** 回到你手里由你 review、merge、部署。它默认你有本地环境、会用 git、看得懂报错。

这一篇换一整类读者——**不写代码的人**。一个开餐馆的老板想要个在线订座页,一个市场同学想要个活动报名表 + 后台。他们**没有本地环境、不会 `npm install`、不知道数据库连哪、更不会部署**。他们要的不是一段 diff,是一个**打开就能用、有链接能发出去的应用**。

这就是 **Vibe Coding(氛围编程)** 这一类工具的战场:

```
   编码 agent(前面几篇)                Vibe Coding 应用生成器(本篇)

   读者:会写代码的工程师               读者:不(太)会写代码的人
        │                                    │
        ▼                                    ▼
   在【你的】仓库里改代码              一句话 →【它】整包生成前端+后端+DB
        │                                    │
        ▼                                    ▼
   交付一个 diff / PR                  交付一个【已部署、有 URL】的活 app
        │                                    │
        ▼                                    ▼
   你负责:环境 / git / 部署           它负责:环境 / 数据库 / 密钥 / 上线
   ────────────────────────           ────────────────────────────
   「帮我写这段代码」                  「帮我把这个想法变成能用的东西」
```

术语上,人们把「不精确描述、靠感觉迭代、代码细节交给 AI」这种造软件的方式叫 **vibe coding(氛围编程)**——2025 年前后火起来的说法。本篇拆的四家,就是把这件事产品化做得最狠的代表:**Replit Agent、Bolt.new、Vercel v0、Lovable**。

> 读这一篇的正确姿势:**别把它们当「更傻瓜的 Cursor」。** 它们和编码 agent 的分野不在「谁更聪明」,而在**交付形态**——编码 agent 交 diff 给工程师,这一类交**已托管上线的整个 app** 给非工程师。这个差别会一路决定它们的架构:执行环境、上下文、验证、部署,处处不同。

## 1. 这一类到底是什么、边界在哪

先给一句话定义:

> **Vibe Coding 应用生成器 = 面向(半)非专业用户、把「一句话需求」整包变成「已联好数据库、经过自测、一键部署上线」的全栈 web app 的垂直产品。** 它不是给你搭 agent 的框架,而是把 agent + harness + 执行/部署基础设施**焊成一个闭环产品**。

拆开这句话里的几个关键基因,顺便和前面几篇划清界:

- **面向非专业用户**:它默认你**不会**配环境、不会 git、看不懂报错,所以必须把这些全藏起来,用「聊天 + 实时预览」当唯一界面。编码 agent 默认你是工程师,这一类默认你不是——**这是最根本的一条分野**。
- **整包生成**:交付物不是「一段建议」或「一个补丁」,是**前端 UI + 后端逻辑 + 数据库 + 鉴权 + 部署**一条龙。你要的是「能用的东西」,不是「一块代码」。
- **托管执行 + 托管部署**:代码在**它的环境里**跑(浏览器沙箱 / 云容器 / 云 VM),生成完直接给你一个**能访问的 URL**。你不碰服务器,也不碰自己的机器。
- **垂直整合、不可移植**:和讲 Claude Code、讲 LangGraph 那几篇里「能装到你机器上」的可移植 harness 不同,这四家的 agent 层、harness 层、平台层**深度绑死在自家云**(或自家浏览器运行时上),**拆不出来单独用**。你订阅的是一整个产品,不是一个库。

为什么值得单独一章?因为**「交付一个活 app 给不写代码的人」这个约束,倒逼出一套编码 agent 里根本不会出现的设计**:执行环境要么塞进浏览器、要么每人一个云沙箱;验证不能只跑 `pytest`,得**真的去点按钮**看功能是不是活的;上下文里除了代码还得塞进「这个库有哪些表」;部署得是**一键、零配置、自带域名**。这些约束是编码 agent 没有的。

> 一句话:**编码 agent 的读者会 review 那个 diff;Vibe Coding 的读者只会看那个 URL 打开对不对。** 谁来兜「看不见的代码里有没有坑」,就成了这一类最要命的问题(见 §5)。

## 2. 一次「造 app」在它体内怎么流动

四家实现差异很大,但「一句话进去、一个上线的 app 出来」的**生命周期骨架是通用的**。先把这条通用流程看透,再看各家在哪一环下重注。设想一个非工程师丢进去一句:

> 「帮我做一个宠物寄养的预约页:用户能选日期、填联系方式、后台能看到所有预约。」

一个 Vibe Coding 生成器内部大致这样走一大圈:

```
  一句话需求
     │
     ▼
① 起环境    provision 一个执行环境(浏览器内 WebContainer / 云容器 / 云 VM),
     │      往往顺手起一个开发用数据库
     ▼
②(可选)规划  先出一个方案/任务分解,给你看、等你点头,再动手写代码(Plan Mode)
     │
     ▼
③ 组上下文   选相关文件(不是塞整库)+ 塞进「有哪些表、有什么规则」+ 系统提示
     │
     ▼
④ 生成       模型流式吐出「要写哪些文件、跑哪些命令、建哪些表」
     │
     ▼
⑤ 落地执行   在环境里真的写文件、装依赖、起 dev server、建库表 → 出实时预览
     │
     ▼
⑥ 自测/验证  ★ 真的去点按钮、填表单、查数据库,判断「功能是不是真的活的」
     │
     ├──❌ 有问题 ──▶ 把报错/失败塞回去 → 回到 ④ 自纠,再来一圈
     │
     ▼
   ✅ 看起来对了
     │
     ▼
⑦ 部署上线   一键打包 → 托管到自家 CDN/云 → 给你一个能访问的 URL
```

注意 ⑥ 那个**验证回环**——它是这一类最有辨识度的心脏。编码 agent 验证靠「跑测试套件」;但非工程师的需求里根本没有测试,而且「按钮渲染出来了」不等于「点了真能预约」。所以这一类必须自己**造一个真实交互**去验证:打开浏览器、点、填、查库,看功能到底活没活。Replit 官方给这个失败模式起了个很传神的名字——**Potemkin 界面(波将金界面)**:看着能用(按钮画出来了、仪表盘有数),其实**没有事件处理、数据是 mock、链接是死的**。⑥ 就是专门来拆穿波将金界面的。

四家在这条骨架上的**分野**恰好落在不同环节,挑两个走一遍最能看清:

- **Bolt.new——把整条流水塞进你的浏览器 tab。** 它最激进的选择在 ①:执行环境不在云上,而是一个跑在**浏览器里**的虚拟 Node.js(WebContainer)。于是它的回路是「模型流式吐出一段自定义 XML → 边流边解析边在浏览器里执行 → 报错当场回灌」,整个过程**服务端零执行**——用它工程师的话说,「服务器就是你的浏览器」。
- **Replit Agent——把 ⑥ 验证做成招牌。** 执行环境是云上一个 Docker 容器,但它真正的重头戏在 ⑥:Agent 会**周期性地决定「我该测一下了」**,在面板里开浏览器预览,你能**亲眼看到它的光标在你的 app 里点来点去**——填表、翻月历、查 API。它不是看像素,而是读增强版 DOM + 直接查库 + 读日志,判断功能真伪。这套自测把「无人干预能连续工作」的时长从约 20 分钟拉到 200+ 分钟(官方口径,见 §3.1)。

> 一句话:**这一类的本事,全在「⑥ 那个『去点一下看看真的活没活』的回环里」。** 谁把验证做得越真,谁生成的就越不是波将金界面——这也是它们和「只会写代码、不会验功能」的编码 agent 最大的手艺差别。

## 3. 拆开看

现在把四家搬上手术台。**Replit Agent、Bolt.new 深拆**(各自一步步讲控制回路与核心组件),**v0、Lovable 做群像横扫**(点定位/独特点/开源闭源/状态)。

### 3.1 Replit Agent:多 agent 编排 + 「code-as-tool」 + 自测穿透波将金

**它是谁**:Replit Agent 是 Replit(CEO Amjad Masad)的**闭源专有 SaaS**,一个官方口中「action-oriented(动作导向)」的自主编码 agent——你一句话,它自己**规划 → 写代码 → 配基建(数据库/密钥/部署)→ 自测 → 交付**一个能发布的全栈应用。版本线:Agent v1(2024)→ **Agent v2**(2025-02,配 Claude 3.7 Sonnet)→ **Agent 3**(2025-09,当前主力)。它无公开源码仓库,**没有 star 数可谈**。截至 2026 年中势头极猛:2026-03 完成 $400M D 轮(Georgian 领投),估值 **$9B**,平台用户 50M+(部分为公司自述)。它**不训练自己的前沿模型**,而是调第三方前沿模型并**按难度路由**(官方有 Lite/Economy/Power+Turbo 的 effort 档,高档把复杂任务送去 frontier models;2026 年具体跑哪几个模型的组合只见二手来源、官方未确认,这里不写死)。

它值得记住的是**三个组合拳**:

**招牌一:多 agent 编排,而且刻意「不追求全自主」。** 它从早期单 agent ReAct 演进成一个分工小队:**Manager agent** 统筹、把目标拆成**尽可能小**的子任务;**Editor agent(s)** 执行编码(任务越小、单 agent 出错率越低);**Verifier agent** 校验代码,但有个**反直觉的独门行为**——它**经常主动退回来找用户对话**,而不是自动往下推。这个「刻意不全自动」是 Replit 明确的哲学:**「不追求 full autonomy,要用户持续参与」**。这和讲自主 SWE agent 那篇里 Devin「派出去就走」的极端自主,是**相反的下注**——因为它的读者是非工程师,一个跑偏两小时、烧一堆钱、交出个错东西的 agent 对不会 review 代码的人是灾难,所以它宁可多问几句。

**招牌二:不用 function calling,让模型「写代码来调工具」。** 它让模型**生成代码**调工具,并写了一套**受限的 Python DSL** 承接(工具库 30+,调用成功率约 90%)。为什么?因为**代码天生会组合**——比如自测要「把日历往后翻 36 个月」,写成**一个循环**就行,而不是发 36 次独立工具调用,token 和成本都省。这和讲自主 SWE 那篇里 OpenHands「动作即代码(CodeAct)」是同一路思想:代码是最通用的动作空间。

**招牌三:REPL-based 自测,专治波将金界面(Agent 3 头牌)。** 前面 §2 那个「光标在 app 里点来点去」的画面,拆开看是这样:

- 它在沙箱的 REPL 里**执行注入了 Playwright 辅助函数的 JS**,用「**code use** 而非 **computer use**」去点按钮、填表单、查 API,甚至能用 Replit Auth 自动登录测登录流;
- 判断依据不是像素,而是**增强版 DOM(带 ARIA label、test 属性)+ 直接查库的工具 + 前后端日志**——所以它能分辨「按钮画出来了」和「点了真有反应」;
- 测试**跑在独立子 agent 里**,不污染主 agent 上下文(主 agent 上下文约 80k–100k token);
- 测完回一份**测试摘要 + 自动修复**发现的问题,再回去重跑直到通过。

官方数字(均为**厂商自报口径**,独立基准未公开):自测把无人干预时长从 **20 分钟拉到 200+ 分钟**;单次测试**中位成本约 $0.20**;整体自称比通用 Computer Use **快约 3×、便宜约 10×**,自主成功率约 90%。

其余组件对着 §2 生命周期扫一遍:

- **执行环境**:每项目一个 **Repl = Docker 容器**(非 root `runner` 用户),用 **Nix** 管系统包与运行时;数据由 **bottomless storage**(NBD 虚拟块设备,后端 GCS,16 MiB 不可变分块 + manifest)承载。
- **快照引擎 + 检查点**:每完成一阶段自动 commit——代码侧一个 git commit(每 app 有不可变 append-only 的 git remote),数据侧一次 **Copy-on-Write 快照**。于是你能**回滚到任意历史检查点**(这也是**计费单位**);CoW「remix」还让 agent 能在隔离副本里试危险操作再随时回滚——这是纯本地/纯 IDE agent 没有的基建级安全网。
- **规划**:显式 **Plan Mode**——先出「有序任务分解 + 架构取舍」,**写任何代码前让你批准**,防跑偏;还有 Design Canvas 边建边渲染真实界面。
- **只碰 dev 库**:agent **只对开发数据库**有权,生产库物理隔离。
- **Agent 3 特有**:你能用自然语言让它**生成「别的 agent / 自动化」**(Telegram bot、Slack agent、定时任务),自动接第三方连接器(Notion / Linear / GitHub / Google Drive 等)并配好凭据。
- **可观测性**:内部用 **LangGraph** 编排 + **LangSmith** 观测 trace。

> 一句话记住 Replit Agent:**它把「agent 造 app」做成了平台级垂直整合——多 agent 分工 + code-as-tool + 真浏览器自测 + CoW 快照回滚——而且它把『刻意不全自主、要人一直在环』写成了明确哲学(Lovable 也收着,但走的是另一条路,见 §3.3)。**

### 3.2 Bolt.new:把整个 Node.js 塞进浏览器,单 artifact 边流边执行

**它是谁**:Bolt.new 是 **StackBlitz**(CEO Eric Simons)2024 年 10 月推出的「浏览器内全栈 AI app builder」。护城河一句话说不清,但先记住这个反直觉的事实——**它不用云端 VM、也不用你本地环境,整个 Node.js 跑在你的浏览器 tab 里**。它是**闭源产品**,但有两个开源亲戚容易混:StackBlitz 2024 年把 bolt.new 的**前身/简化版**以 MIT 开源在 `stackblitz/bolt.new`(约 16.4k star,是历史快照、非生产版),真正活跃的开源分支是社区维护的 **`stackblitz-labs/bolt.diy`**(MIT,约 19.4k star,支持 19+ 家 LLM 可插)。注意:**底层 WebContainer API 本身不是 MIT**,商用生产要单独授权。截至 2026 年中活跃增长(ARR 从首周约 $1M 冲到 2025-03 约 $40M),近期有 Bolt Cloud(2025-08)和 2026-05 与微软合作(Azure 部署 + 上架 Marketplace)。它**深度绑定 Anthropic Claude**:发布起就独家用 Claude(launch 对应 2024-10 升级版 Claude 3.5 Sonnet),到 2026 年默认代理升级为「**Claude Agent**」(基于更新的 Claude Sonnet 系列;确切型号官方未逐版点名),同时正在退役自研旧「v1 Agent」(2026-04 起新项目不可选,2026-08 后自动迁移)。

它值得记住的是**两个子系统**:一个是护城河,一个是控制回路。

**护城河:WebContainer——浏览器里的一整个操作系统。** 别家把执行放云端容器/VM,Bolt 把整个 Node.js 环境放进**浏览器的 WASM 沙箱**。据 PostHog 的工程报道,它内部是一个 **Rust 写、编译成 WebAssembly 的微内核 OS**(管进程),一个 Rust→WASM 的**虚拟文件系统**(用 `SharedArrayBuffer` 让多 Web Worker 并发,实测比 IndexedDB/OPFS 快),**每个 Web Worker 当一个 Node 进程**,一个自研 TypeScript shell **JSH** 模拟 Bash,以及一个**虚拟网络栈**(用 Service Worker 拦截内部 URL 映射到浏览器、WebSocket 桥做热更新、数据库客户端走 TCP tunnel 兜底)。性能上运行时 bundle < 10 MB、`npm install` 常 < 500ms。

**安全模型也因此变了**:100% 代码执行在浏览器安全沙箱内——**没有云 VM、不碰你本地文件系统**,这是它和所有「云端沙箱」竞品最根本的隔离差异。代价是硬的:WebContainer **跑不了原生二进制**(无 C/C++ 编译、无 g++)、**没有 pip**(Python 仅限标准库)、**Git 不可用**。这些约束直接写进了系统提示——比如硬性要求「优先 Node.js 脚本而非 shell 脚本」。

**控制回路:不用 function calling,用自定义 XML 动作协议,而且『单 artifact、边流边执行』。** 模型不吐 JSON 工具调用,而是被系统提示要求吐一段 XML:`<boltArtifact id="..." title="...">` 包裹**整个方案**(一项目一 artifact,改动时复用同一 id 做增量),里面若干 `<boltAction>`——`type="shell"`(跑命令)、`type="file" filePath="..."`(写完整文件)、`type="start"`(起 dev server)等。系统提示几条硬规则很说明它的脾气(逐条来自 `prompts.ts`):**「先 HOLISTICALLY & COMPREHENSIVELY 地想清所有文件/依赖再建 artifact」**、**「永远先装依赖再生成别的」**、**动作必须有序**(文件得先存在才能跑执行它的命令)、**「绝不用占位符」**、**「别啰嗦,除非用户问否则啥都别解释」**。

它一次请求的回路(组件名多取自开源的 bolt.diy,生产版可能已演进,已如实标注):

```
用户 prompt
   │
   ▼
① 输入富化      客户端把附件/模板/当前工作区上下文拼进消息
   │
   ▼
② 服务端上下文优化  /api/chat 从工作区【选相关文件】(不是塞整库)+ 拼系统提示
   │
   ▼
③ 流式调模型     streamText() 调 Claude,SSE 一块块流回
   │
   ▼
④ 边流边解析     StreamingMessageParser 从 token 流里实时抠 <boltAction>
   │              ——【解析出一个动作就入队一个,不等整条消息】
   ▼
⑤ 串行执行       WorkbenchStore 建 ActionRunner,全局队列【顺序】跑、文件锁防竞态
   │              写文件→同步编辑器;shell→JSH;start→npm install→spawn dev server
   ▼
⑥ 实时反馈闭环   文件树/终端/预览/diff 实时更新;报错被捕获成 alert
   │              ——用户或代理可把报错回灌聊天,触发下一轮修复
   ▼
⑦ 长响应续写     撞 token 上限没写完 → CONTINUE_PROMPT 从断点续,历史落 IndexedDB
   │
   ▼
⑧ 部署          一键部署(Netlify / Bolt Cloud)
```

注意这是个**单代理**架构——没有 Manager/Editor 分工,所有工作在一个执行队列里**串行**跑,这是刻意取舍:换来**可预测、少竞态**。规划也没有独立 planner,被压进「一次 holistic 思考 + 一个大 artifact 里排好有序动作」。人在环是宽松式的——**看实时预览 + 报错回灌 + 用下一句 prompt 修正 / rewind**,不像某些工具那样每个动作逐条审批。

> 一句话记住 Bolt.new:**它把执行环境从云上搬进你的浏览器(WebContainer),把动作从 JSON 换成流式 XML(单 artifact 边流边执行),用『首字节到可预览更快』换来了最丝滑的 vibe coding 体感——代价是跑不了原生二进制、pip、真 Git。**

### 3.3 群像横扫:v0 与 Lovable

深拆了「多 agent + 云容器 + 真自测」的 Replit 和「浏览器运行时 + 单 artifact」的 Bolt,再横扫两家同赛道、但把重注下在别处的产品。看它们各自赌了什么。

**Vercel v0——护城河不在基座模型,在『复合模型』的确定性夹层。** v0 是 Vercel(Next.js 母公司)出品的生成式 UI / 全栈 web app builder,专造 React / Next.js + Tailwind + shadcn/ui 生态。2023 年首发时只是「输入提示 → 拿一个 React 组件」;2025-08 `v0.app` 扩成「任何人用提示造并部署 app」的全栈产品;2026-02「the new v0」引入 sandbox 运行时、可导入任意 GitHub repo、每个 chat 建分支/开 PR/合并即部署。**产品本身闭源**,但官方 SDK `v0-sdk` 是 **Apache-2.0 开源**(约 500+ star,只是 SDK 仓库、不代表产品用户量)。它最值得记的独特点:Vercel 把 v0 明确称为 **composite model(复合模型)**——不是「prompt 套壳一个基座」,而是把可靠性做成**「确定性工具 + 小快模型」的后处理夹层**:前沿推理**外包给 Claude**(`v0-1.5` 系列基座是 Claude Sonnet 4),但护城河在夹层的三个流水线组件——**Dynamic System Prompt**(用 embeddings + 关键词判意图、按需注入当前版本 API 知识防幻觉,且保持稳定吃 prompt-cache)、**LLM Suspense**(在 token 流向用户的**途中**改写:修错误 import、把图标名放向量库 ~100ms 内纠到真实存在的 `lucide-react` 名)、**Autofixers**(确定性补依赖 + 一个微调小模型 `vercel-autofixer-01` 解析 AST 边流边修错,<250ms 按需触发)。官方自报基准(**Vercel/Fireworks 自报、非独立评测**)称 `v0-1.5-md` 无错生成率 93.87%,显著高于同基准的裸基座。截至 2026 年中活跃、是 Vercel 旗舰 AI 产品。**记住 v0 的一句话:它赌「可靠性来自基座外面那圈确定性夹层,而不是更强的基座」——这正是讲 harness 那几篇「harness 比模型更重要」的一个极端工业样板。**

**Lovable——反多 agent、意见极强的固定技术栈、把智能集中在生成这一步。** Lovable(瑞典斯德哥尔摩,创始人 Anton Osika,前身是曾经 GitHub 增长最快仓库之一的开源项目 **GPT Engineer**)是一个用聊天造全栈 web app 的 vibe-coding 平台——前端 UI、数据库、鉴权、支付、部署一条龙。**产品本体闭源**;开源的只有前身 `AntonOsika/gpt-engineer`(MIT,约 55.2k star,已于 2026-04 归档只读)。它是 **Anthropic 官方客户案例主角**:推理全调 Claude API,官方骨架措辞是「a harness around a main agent that can use subagents … with the right models at each step(围绕主 agent 的 harness,主 agent 用子 agent 编排、每一步配最合适的模型)」。它最值得记的独特点有两个,而且都**逆当时的社区潮流**:其一,**刻意避开复杂多 agent 架构**(据第三方拆解,理由是多 agent 准确率更低、失败更难解释、更慢),转而「把智能集中在生成这一步」——用廉价模型做路由/选文件、强模型做生成、**验证交给非 LLM 的确定性工具**(真实浏览器点击/填表/截图、Vitest 前端测试、edge function 调用测试)。其二,**一套意见极强的固定技术栈**(React + TS + Tailwind + shadcn/ui,后端一律 **Supabase**),信条是「什么都能生成的 agent 会生成不一致的东西;永远输出同一栈的 agent 才可被评测、调优」——**牺牲通用性换可靠性与可评测性**。它的「记忆」用 Git commit 历史充当(每次编辑一个原子 commit),对外还暴露一个 **Lovable MCP server** 让 ChatGPT/Claude/Cursor 等外部 agent 来调度建/部署项目。截至 2026 年中增长极快:2025-12 B 轮 $330M(估值 $6.6B),2025-11 达 $200M ARR,官方称约 8M 用户、日新建约 10 万项目。(注:其 harness 内部机制——三层上下文、两阶段选文件、Haiku/Sonnet/Opus 分层路由、「降错约 90%」——**主要来自单一第三方技术拆解**,官方只确认了「harness + 主 agent + 子 agent + 按步选模型 + 每次新模型做回归评测」这层骨架,细节按参考看。)**记住 Lovable 的一句话:它和 Replit 一样反「越自主越好」,但反的方式相反——不是靠多问用户,而是靠『固定技术栈 + 反多 agent + 确定性验证』把不确定性从根上摁掉。**

## 4. 它们与众不同的设计决策与取舍

退一步看,四家在回答同一个问题——**「怎么让一个不写代码的人,从一句话稳稳拿到一个能上线的 app?」**——但下注的位置完全不同。四条主线:

**第一条:执行环境放哪?(分野最大的一维)** Bolt 押「浏览器」(整个 Node.js 塞进 WebContainer,服务端零执行、毫秒级启动,代价是跑不了原生二进制/pip/真 Git);Replit 押「云容器」(Docker Repl + Nix + CoW 快照,能跑真环境、能安全试危险操作,代价是重、锁在自家云);v0 / Lovable 押「云沙箱 + 托管后端」(v0 绑 Vercel 部署链路,Lovable 绑 Supabase)。

**第二条:靠什么把「看不见的代码」变可靠?** 四家四个答案——Replit 靠**真浏览器自测**(穿透波将金界面),v0 靠**确定性后处理夹层**(流内修错 + AST autofixer),Lovable 靠**固定技术栈 + 确定性验证**(同一栈才可评测),Bolt 靠**实时预览 + 报错回灌**。但**没有一家是靠『训一个更强的模型』**——它们全外包 Claude(或多模型路由),把功夫下在模型外面。

**第三条:自主 vs 人在环?** 这一类整体**比自主 SWE agent 更收着**——读者不会 review 代码,放它跑飞的代价更大。Replit **明确「不追求 full autonomy」**、Verifier 主动回来问、Plan Mode 事前批准;Lovable 的 Plan Mode 也要你点头才执行。它们把「人在环」当特性,不当负担。

**第四条:动作协议——JSON 还是别的?** 四家里有两家**都躲开了标准 function calling**:Replit 用**受限 Python DSL(让模型写代码调工具)**,Bolt 用**自定义 XML 动作协议(单 artifact)**。理由相同——**代码/结构化协议天生会组合、能边流边执行、能表达「有序步骤」**,比一串孤立 JSON 调用更适合「一次生成一整个 app」。

一句话把四条对齐:**Bolt 改造 app 在哪里『跑』,Replit 改造 agent 怎么『自测』,v0 改造模型输出怎么『修』,Lovable 改造 agent 该不该『分身』。**

> 一个贯穿全篇的反直觉:**这四家没有一家赢在模型上——它们全在用同一批前沿模型(基本都是 Claude)。** 拉开差距的是「模型外面那圈东西」:执行环境、自测方式、修错夹层、技术栈约束、部署闭环。这又一次印证了讲 harness 那几篇的核心论点——**发动机大家都买同一台,能不能把一句话稳稳变成上线的 app,拼的是发动机外面那套 harness 与基础设施。**

## 5. 局限与坑

这一类是 2025–2026 最被神化的赛道,也最容易踩坑。诚实列几条(机器会渲染成 ⚠️ 警示卡):

1. **波将金界面:「能预览」不等于「真能用」。** 这是这一类的**头号原生失败模式**——按钮渲染了、仪表盘有数,但**没有事件处理、数据是 mock、链接是死的**(Replit 官方原话)。为什么致命:读者**不会读代码**,只会看预览,预览好看就以为成了,一发给真实用户当场穿帮。怎么避:优先选**真的会去点按钮、查数据库**做验证的工具(Replit 的 REPL 自测、Lovable 的确定性浏览器测试),别只信截图;关键流程(提交、支付、登录)自己也务必亲手点一遍。

2. **数据库和密钥的安全,才是真正的爆炸半径。** 它替你联好了库、配好了鉴权、藏好了密钥——但你**看不见这些配置对不对**。以 Supabase 系(Lovable 等)为例,**RLS(行级安全)策略配错是主要漏洞面**:一个错策略可能让任何人读到全部用户数据。为什么致命:非工程师根本意识不到「这里有个安全边界」。怎么避:把「谁能读写哪张表」当成必须人工确认的一环;选**把安全扫描做进核心回路**(密钥检测、RLS 分析、依赖扫描)的工具,而不是事后补。

3. **厂商自报的数字,基本都没独立审计。** 「自主成功率 90%」「快 3×、便宜 10×」「无错生成率 93.87%」「降错约 90%」——**几乎全是厂商(或合作方)自报口径,基准细节多未公开、无第三方复现**。为什么致命:拿两家裸数字直接排名,等于拿两把不同刻度的尺子比长短。怎么避:引用任何数字都带上「谁报的、测的什么、什么时候」,当「营销声称」读——本篇给的所有数字都标了来源和「厂商口径」。

4. **平台锁定 + 不可移植,是垂直整合的原罪。** 它把 agent + harness + 数据库 + 部署全焊死在自家云,换来「开箱即得一个上线的 app」,代价是**你几乎搬不走**。为什么致命:等 app 长大、要接企业级需求(网络隔离合规、dev/staging/prod 环境、精细成本可见性),这些平台往往**给不了**(ml6 对 Lovable 的企业边界分析点了这几条)。怎么避:把它当「原型加速器 / 内部工具生成器」用,别一上来就押上要长期演进的核心业务;需要时用双向 GitHub sync 把代码接给真人工程师。

5. **增量 prompt 会累积复杂度、丢掉整体结构。** vibe coding 的爽点是「不满意再说一句改一改」,但一句句叠加改,代码结构会**越滚越乱**——agent 每次只看局部,慢慢丢掉全局设计。为什么致命:项目一大,后期一个小改动可能触发一连串莫名其妙的连锁 bug,而你没能力 debug。怎么避:复杂项目**先用 Plan Mode 定好架构再动手**(Replit、Lovable 都有);阶段性用检查点/回滚,发现越改越乱就退回干净的检查点重来。

6. **别把「有了 URL」等于「做对了、也安全了」。** 它给你一个能访问的链接,只说明「它跑通了、预览打开了」。**它没测到的边界、没想到的鉴权、没考虑的滥用场景,照样是坑。** 为什么致命:一个对公网开放的 app,承载的是真实用户的真实数据。怎么避:上线前把关键路径(登录、权限、支付、数据可见性)当成**必须人工过一遍**的清单——这一类把你从「写代码」解放到「点点看」,但**没把你从『对上线的东西负责』里解放出来**。

## 6. 一句话记住 + 对比表

> **Vibe Coding 应用生成器 = 面向不写代码的人、把一句话整包变成『已联库、经自测、一键上线』的全栈 app 的垂直产品。Bolt 把执行搬进浏览器,Replit 把自测做成招牌,v0 把可靠性做成确定性夹层,Lovable 用固定技术栈反多 agent——四条路没有一条靠更强的模型赢,拼的全是模型外面那圈 harness 与基础设施。**

四家横向对齐(数字均为**截至 2026 年中的厂商/报道口径快照**,会持续漂移):

| 维度 | Replit Agent | Bolt.new | Vercel v0 | Lovable |
|---|---|---|---|---|
| **定位** | 平台级自主编码 agent(整包造 app + 一键部署) | 浏览器内全栈 app builder | 生成式 UI / 全栈 builder(React/Next 生态) | 聊天造全栈 app 的 vibe-coding 平台 |
| **内核(招牌)** | 多 agent(Manager/Editor/Verifier)+ code-as-tool DSL + 真浏览器 REPL 自测 + CoW 快照 | WebContainer(浏览器内 Node)+ 单 artifact XML 边流边执行(单代理) | Composite model:基座 Claude + 确定性修错夹层(Dynamic Prompt / LLM Suspense / Autofixer) | 围绕主 agent 的 harness + 子 agent 按步选模型 + 反多 agent + 固定技术栈 + 确定性验证 |
| **执行环境** | 云端 Docker Repl(Nix + bottomless storage + 快照引擎) | **浏览器内 WebContainer(WASM 沙箱,无云 VM)** | sandbox 运行时(可导入任意 GitHub repo) | 虚拟浏览器测试 + Supabase 后端 + CDN 部署 |
| **与模型关系** | 不自训;多模型按 effort 路由(调 frontier models) | 不自训;深绑 Anthropic Claude(「Claude Agent」) | 不自训;基座 Claude Sonnet(仅小修错模型自训) | 不自训;全调 Claude API(Anthropic 客户案例) |
| **开源/闭源** | 闭源专有 SaaS(无 star) | 闭源;前身 `stackblitz/bolt.new`(MIT,~16.4k★)+ 社区版 `bolt.diy`(MIT,~19.4k★) | 产品闭源;`v0-sdk` Apache-2.0(~500+★) | 闭源;前身 `gpt-engineer`(MIT,~55.2k★,已归档) |
| **自主 / 人在环** | 明确「不追求全自主」;Verifier 主动问 + Plan Mode 批准 | 宽松:实时预览 + 报错回灌 + rewind | 人在环主要靠 Git 面板(PR/合并即部署) | Plan Mode 需批准 + Visual Edits 直接改 |
| **最适合谁** | 想从一句话拿到上线全栈 app、还想生成 bot/自动化的非/半专业用户 | 想在浏览器里秒起原型、要毫秒级启动/离线、不需原生依赖的人 | 专造 React/Next UI、深用 Vercel 部署链路、要高一次成型率的人 | 要 React+Supabase 全栈闭环、一天内从想法到可收费上线的人 |
| **2026 状态** | 活跃高增长(Agent 3;估值 $9B) | 活跃(v1 Agent 退役中→Claude Agent;联微软) | 活跃(Vercel 旗舰 AI;the new v0) | 活跃极快增长(B 轮 $6.6B;$200M ARR) |

**怎么选**:想要**执行零服务端成本、毫秒级起原型、能离线**,且不碰原生依赖 —— 看 **Bolt.new**;想要**从一句话拿到真正上线的全栈 app、还想让它自测穿透波将金界面、甚至生成 bot** —— 看 **Replit Agent**;**深在 Vercel/Next 生态里、要最高的一次成型率与最少的代码错误** —— 看 **v0**;想要 **React + Supabase 一条龙、一天内从想法到可收费上线** —— 看 **Lovable**。

对照前面几篇:讲 Claude Code、Cursor、Aider 的那几篇拆的是**帮工程师写代码**的编码 agent,交付一个 diff;讲自主 SWE agent 的那篇把「派出去自己跑」推到极致,但读者还是工程师、终点还是 PR。**这一篇补上了另一半天空——面向不写代码的人、交付一个已上线 app 的整包生成器。** 它们和编码 agent 共享底层模型,却因为「读者不会 review 代码」这一条约束,长出了一整套不同的执行、自测、部署机制。
