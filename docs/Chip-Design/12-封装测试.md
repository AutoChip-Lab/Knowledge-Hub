# 芯片设计完整流程 · 第 ⑫ 步:封装测试

> 「芯片设计完整流程」学习系列第十二篇。
> 十三个编号主阶段:① 产品/市场需求 → ② 系统规格 → ③ 架构 → ④ 微架构 → ⑤ RTL → ⑥ 验证
> → ⑦ 综合 → ⑧ 物理设计 → ⑨ 签核 → ⑩ 流片 → ⑪ 制造 → ⑫ 封测 → ⑬ bring-up/量产。
> DFT设计涉及:③ 测试架构与覆盖率目标 → ④/⑤ 测试控制和可测逻辑 → ⑦ Scan/压缩/MBIST 插入与 ATPG → ⑧ Scan 链重排及测试时钟、功耗、拥塞收敛 → ⑨ 测试模式与覆盖率签核 → ⑫/⑬ ATE、诊断与良率学习。

---

## 0. 先看全景:芯片从想法到出片,大致这几步

```
① 产品/市场需求 → ② 系统规格(Spec) → ③ 架构设计 → ④ 微架构设计 → ⑤ RTL 设计
   → ⑥ 功能验证 → ⑦ 逻辑综合 → ⑧ 物理设计(布局规划/布局/时钟树/布线)
   → ⑨ 签核(STA/功耗/EM-IR/物理验证/等价性) → ⑩ 流片(tape-out)
   → ⑪ 制造(Fab) → ⑫ 封装测试 → ⑬ 芯片回来后 bring-up/量产
DFT设计:③ 测试架构 → ④/⑤ 测试控制和可测逻辑 → ⑦ Scan/压缩/MBIST 与 ATPG → ⑧ Scan 链重排/测试时序功耗 → ⑨ 测试模式与覆盖率签核 → ⑫/⑬ ATE/诊断/良率学习
```

**封装测试夹在「制造(⑪)」和「量产/bring-up(⑬)」之间**:制造出的是一整片晶圆,上面是裸 die——又脆弱又没法直接用。封测做两件事:**封装**(把 die 切下来、装进保护壳、连上能与外界通电的引脚/球)+ **测试**(对每一颗芯片做电学测试,筛掉制造缺陷)。它把"一片硅"变成"能插到板子上、且保证是好的"成品。

> 一句话定位:**封装让芯片"能用"(有壳、有引脚、能散热),测试让芯片"可信"(逐颗筛掉坏的)——只有又能用又可信的,才算交付得出去。**

---

## 1. 封装测试到底是什么

一句话:**把晶圆上的 die 切割、封进封装、连好 IO 与供电、做好散热,并对每一颗做电学/功能测试,筛除制造缺陷,最终交付"已知好芯片(KGD)"。**

### 两条线:封装(让它能用)+ 测试(筛出好的)
- **封装(package)**:die 太小太脆、引脚太细,无法直接使用。封装提供**机械保护、电气连接(引脚/焊球)、散热路径**,把纳米级的 die 接到毫米级的板子世界。
- **测试(test)**:制造有缺陷,必须**逐颗测**——量产测试(⑫)和功能验证(⑥)是两码事:⑥ 验"设计对不对",⑫ 筛"这颗造得好不好"。

*在 TPU 上*:制造好的 TPU die 被切下、用倒装(flip-chip)等方式封装、连到 PCIe 板卡能用的封装基板上,做散热;再用 ATE 逐颗测试,筛掉坏片,只把好的装成数据中心能用的加速卡。

---

## 2. 谁来做 + 需要什么输入

**谁来做**:**封装测试厂(OSAT,如日月光/Amkor 等)或 foundry 的封测线**;设计侧由**封装 / 测试工程师**(测试程序、测试结构、封装设计)对接。
**需要什么输入**:**完工晶圆(⑪)**、**封装设计**(基于 ③/⑧ 的封装规划)、**测试程序与图样**(由 DFT设计形成,并随网表、版图和 ATE 调试持续收敛:scan/ATPG、MBIST/JTAG 图样、波形与覆盖率报告)、**ATE 测试机与探针卡**。

> 关键心态:**DFT 在这里"兑现"**——上游没有把 scan/压缩/MBIST、最终图样和测试协议做扎实,这里就测不充分、筛不干净;测试质量直接决定交付质量与现场失效率。

---

## 3. 封装测试的核心内容(尽量穷尽地列)

每项给:**是什么 + 关键点 + 在 TPU 上长什么样**。

### 3.1 封测全景
- **是什么**:大致流程——晶圆测试(CP)→ 减薄/切割 → 封装 → 成品测试(FT)→(可靠性/老化)→ 分级出货。
- **关键点**:**先在晶圆上测(CP),早筛掉坏 die,别浪费封装成本**;封装后再测(FT)确认封装也好。
- *在 TPU 上*:TPU die 先 CP 粗筛,封进板卡级封装后再 FT 全面测。

### 3.2 晶圆减薄与切割(dicing)
- **是什么**:把晶圆**背面磨薄(backgrind)**便于封装/散热,再用**砂轮/激光沿划片槽切割**成单个 die。
- **关键点**:切割不能损伤 die;薄 die 更脆、利于 3D 堆叠与散热。
- *在 TPU 上*:把 ≤331mm² 的大 die 从 wafer 上完整切下,准备封装。

### 3.3 封装类型(wire-bond / flip-chip / 2.5D / 3D)
- **是什么**:把 die 连到封装的方式——**引线键合(wire-bond)**(细金线连焊盘,便宜)、**倒装(flip-chip)**(die 翻过来用焊球 C4 直接连基板,IO 多、性能好)、**2.5D**(die + HBM 经硅 interposer 共封)、**3D**(TSV 垂直堆叠)。
- **关键点**:高性能/高 IO/高带宽芯片走 flip-chip / 2.5D / 3D;这正是 ③ §3.14 chiplet/封装架构的物理落地。
- *在 TPU 上*:高性能 TPU 用 flip-chip 类封装连大量 IO 与供电;后续带 HBM 的 TPU 走 2.5D 共封。

### 3.4 封装工艺(die attach → bonding → molding → balls)
- **是什么**:**贴片(die attach)→ 互连(wire-bond/bump)→ 塑封/盖板(molding/lid)→ 植球(BGA balls)**,做出能贴到板上的成品封装,并提供散热(散热盖/导热)。
- **关键点**:封装要兼顾电(供电/信号完整性)、热(散热)、机械(可靠)。
- *在 TPU 上*:封成带散热盖、BGA/插卡接口的板卡级封装,满足数据中心散热与供电。

### 3.5 晶圆测试(CP / wafer sort)
- **是什么**:封装前,用**探针卡**在晶圆上**逐 die 探测测试(CP,Chip Probing / wafer sort)**,早筛坏 die、做 wafer map。
- **关键点**:早筛省封装钱;CP 还反馈 wafer 良率与缺陷分布给制造。
- *在 TPU 上*:对 wafer 上每颗 TPU die 先 CP,标好哪些好哪些坏。

### 3.6 成品测试(FT)与 ATE
- **是什么**:封装后做**最终测试(FT,Final Test)**——用 **ATE(自动测试设备)** 跑**测试程序**:scan/BIST 图样、at-speed 功能、IO、参数测试,验证"造得对、跑得到速"。
- **关键点**:测试质量与覆盖率(由 DFT 结构、最终 ATPG/MBIST 图样及参数/功能测试共同决定)影响漏检率;测试时间直接进入单位成本。
- *在 TPU 类 AI 加速器上*:ATE 通常要组合 logic scan、memory BIST、IO/参数与 at-speed/功能图样。TPU v1 的真实量产测试流程未公开,这里不把行业做法当成其产品事实。

### 3.7 老化与可靠性测试(burn-in / reliability)
- **是什么**:**老化(burn-in)**——在高温(常配合升高电压)下**长时间(典型数小时~数百小时)持续运行**,靠加速老化把会**早夭(infant mortality)**的弱片提前逼坏淘汰;以及抽样的可靠性试验。
- **关键点**:对高可靠场景(数据中心、车规)尤其重要——把"出厂就会坏"的提前淘汰。
- *在 TPU 上*:数据中心 7×24 运行,老化筛掉早夭片,降低现场失效。

### 3.8 分级(binning)与良率筛选
- **是什么**:按测试结果**分级(binning)**——好/坏、按速度/功耗/质量分档;最终只出货**已知好芯片(KGD)**。
- **关键点**:binning 让同一批 die 按品质卖不同档(产品差异化);良率筛选直接决定交付量与成本。
- *在 TPU 上*:筛出合格的 TPU 加速卡交付数据中心,坏的/不达标的剔除。

### 3.9 封装设计与芯片-封装-板级协同(package 不只是"装壳",更是一项设计)
- **是什么**:前面 §3.3/§3.4 讲的是 OSAT **怎么把 die 装进封装**(工艺);但**封装本身也是一项设计工程**——**基板/interposer 布线、封装级信号完整性(SI)与电源完整性(PI)、芯片-封装-板(die-package-board)三者协同(co-design)、热设计与散热方案**都要专门做。对一块"die + HBM + 封装 + 板卡 + 散热"整体交付的加速卡,这部分工作量不小。
- **关键点**:
  - **基板/interposer 设计**:flip-chip 的 C4 bump/RDL、2.5D 的硅 interposer 走线(把逻辑 die 与 HBM 的几千根线对接),要满足布线密度、阻抗、串扰。
  - **SI/PI co-design**:高速 SerDes 的封装通道、HBM 并行总线、供电路径(封装 PDN 与片上 PDN 一起看)、电源/地平面与去耦电容都要协同仿真——**片上 IR-drop 签核(⑨)和封装/板级供电是连在一起的**。
  - **热设计**:散热盖(lid)、TIM(导热界面材料)、散热器/冷板的配合;结温 Tj 压在规格内,算力阵列的热点要导得出去。
  - **die-package-board 协同**:pad/ball 分配、层叠规划要三方一起定,改一处常牵动另两处。
- *在 TPU 上*:把 ≤331mm² 的 die 用 flip-chip 接到封装基板、做带散热盖的 BGA 封装,设计封装供电与散热,让它在数据中心服务器里 7×24 满负荷散热不超温;后续带 HBM 的版本还要做 2.5D 硅 interposer 把逻辑 die 与 HBM 共封。

### 3.10 量产测试程序(test program / ATE program)开发
- **是什么**:§3.5/§3.6 的 CP/FT 要"跑测试程序",但这套**测试程序本身要开发**——它是连接"特性化(⑬)"与"量产筛选"的一段独立工程:定**每项测试的 limit / guardband**、组织 **scan/MBIST/功能/参数/at-speed** 各类图样、做 **CP↔FT 相关性**、优化**测试时间**(直接进成本)、调通 **ATE 程序**、上**多 site 并测**、落地 **binning 判据**。
- **关键点**:
  - **limit 从哪来**:由**特性化数据 + 规格**定测试上下限并留 guardband;limit 太松漏检、太紧丢良率。
  - **测试时间 = 钱**:海量 SRAM/逻辑靠 **scan 压缩 + MBIST** 压测试时间;ATE 机时按秒计费,测试程序效率直接决定单颗测试成本。
  - **覆盖率靠 DFT**:能测多干净取决于 scan/MBIST 等结构、最终 ATPG 图样与覆盖率收敛——测试程序把 DFT设计在网表、版图和 ATE 调试中积累并签核的能力**兑现**到具体 ATE 波形、limit 与判定流程;上机发现的问题还会反向推动图样和测试条件迭代。
  - **CP↔FT 相关性 + 多 site**:CP(晶圆)与 FT(成品)判据要相关一致;ATE 上同时测多颗(multisite)提吞吐。
- *在 TPU 上*:基于 TPU 的 scan 链/MBIST 与特性化结果开发 ATE 测试程序——定 700MHz at-speed 与各参数的 limit、把阵列/SRAM 的 MBIST 与 scan 图样组织好、压测试时间、做 CP/FT 相关与并测,最终能逐颗经济地判"好/坏/档位"。

---

## 4. 贯穿始终、必须反复权衡的东西

- **测试覆盖 vs 测试成本**:测得越全越筛得干净,但测试时间/ATE 机时直接进单位成本;要用高效 DFT(scan 压缩、BIST)平衡。
- **早筛 vs 晚筛**:CP 早筛省封装钱;但有些缺陷只有封装后/老化后才暴露。
- **封装等级 vs 成本**:flip-chip/2.5D 性能好但贵;wire-bond 便宜但受限——按产品定位选。
- **漏检率 vs 现场失效**:测试不充分 → 坏片流到客户,现场失效代价远大于多测一会。
- **DFT 的债**:前面没规划好 DFT,这里筛不干净——可测性从架构、RTL、综合、DFT 插入到物理/签核一路累积,封测阶段只能兑现,无法凭空补齐。

> 灵魂句:**封装让芯片"接上世界",测试让芯片"值得信任"——而测试能筛多干净,早在 DFT 规划时就决定了;封测只是把那笔债兑现。**

---

## 5. 封装测试"怎么做"出来的(方法与工具,概念层面)

- **封装**:die attach、wire-bonder / flip-chip bonder、molding、植球、散热盖装配设备;封装设计(基板、热)。
- **测试**:**ATE(自动测试设备)** + **探针卡(CP)/ 测试座(FT)** + **测试程序**(scan/BIST/功能/参数图样)。
- **可靠性**:burn-in 炉、可靠性试验设备。
- **流程**:CP → 切割封装 → FT →(burn-in)→ binning → 出货 KGD。

---

## 6. 封装测试阶段的交付物(交给下游什么)

- **封装好的成品芯片 / 模组 / 板卡**:可以真正使用的形态。
- **测试结果与 binning 数据**:每颗的合格性与档位。
- **已知好芯片(KGD)**:筛选后交付 **⑬ bring-up/量产**与客户。
- **良率/失效数据**:反馈给制造与设计(改良)。

> 封测的产物,是"既能用、又可信"的成品芯片——前面所有努力到这里第一次变成可以交到客户手里的东西。

---

## 7. 容易被忽视、却致命的坑

1. **DFT 没规划好**:scan/MBIST 不足 → 测不充分、漏检 → 坏片流到现场,代价远超多测。
2. **测试时间失控**:海量 SRAM/逻辑没用压缩/BIST → 测试机时爆、单位成本高。
3. **只测功能、不测参数/速度**:漏掉 at-speed/参数缺陷 → 现场偶发失效。
4. **不做老化(该做时)**:早期失效流到客户,高可靠场景翻车。
5. **封装的电/热/机械没兼顾**:供电不足、散热不够、机械应力 → 封装级失效。
6. **CP 早筛缺失**:把坏 die 也封了,白白浪费封装成本。
7. **KGD 标准不严**:binning/筛选标准松,次品出货损害口碑。

---

## 8. 小结 + 下一步

**封装测试 = 把晶圆切成 die、封进能用的封装(连 IO/供电、做散热),并逐颗做电学/功能测试筛掉制造缺陷,交付"已知好芯片(KGD)"。** 它的灵魂是 **"封装让芯片能用,测试让芯片可信"**;最该理解的因果是 **"能筛多干净,早在 DFT 规划时就定了"**;最该敬畏的成本是 **"测试覆盖与测试时间的平衡、以及漏检流到现场的代价"**。

下一步(第 ⑬ 步、也是最后一步)是 **bring-up / 量产**:首批芯片回到实验室,**第一次上电点亮、调试、用真实负载验证、并把良率爬坡到可量产**——设计绕了一整圈,终于在真实硅片上跑起来,走向千千万万颗的量产交付。
