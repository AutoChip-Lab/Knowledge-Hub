# 芯片设计完整流程 · 第 ⑩ 步:流片(Tape-out)

> 「芯片设计完整流程」学习系列第十篇。
> 编号主流程:① 产品/市场需求 → ② 系统规格 → ③ 架构 → ④ 微架构 → ⑤ RTL → ⑥ 验证
> → ⑦ 综合 → ⑧ 物理设计 → ⑨ 签核 → ⑩ 流片 → ⑪ 制造 → ⑫ 封测 → ⑬ bring-up/量产。
> DFT设计涉及:③ 测试架构与覆盖率目标 → ④/⑤ 测试控制和可测逻辑 → ⑦ Scan/压缩/MBIST 插入与 ATPG → ⑧ Scan 链重排及测试时钟、功耗、拥塞收敛 → ⑨ 测试模式与覆盖率签核 → ⑫/⑬ ATE、诊断与良率学习。

---

## 0. 先看全景:芯片从想法到出片,大致这几步

```
① 产品/市场需求 → ② 系统规格(Spec) → ③ 架构设计 → ④ 微架构设计 → ⑤ RTL 设计
   → ⑥ 功能验证 → ⑦ 逻辑综合 → ⑧ 物理设计(布局规划/布局/时钟树/布线)
   → ⑨ 签核(STA/功耗/EM-IR/物理验证/等价性) → ⑩ 流片(tape-out)
   → ⑪ 制造(Fab) → ⑫ 封装测试 → ⑬ 芯片回来后 bring-up/量产

DFT设计:③ 测试架构 → ④/⑤ 测试控制和可测逻辑 → ⑦ Scan/压缩/MBIST 与 ATPG → ⑧ Scan 链重排/测试时序功耗 → ⑨ 测试模式与覆盖率签核 → ⑫/⑬ ATE/诊断/良率学习
```

**流片夹在「签核(⑨)」和「制造(⑪)」之间**:签核通过后,设计就**正式交付给晶圆厂(foundry)**——把最终版图转成可写掩模的数据、做出光掩模(mask),制造正式开工。"tape-out"这个词源自早年把版图数据存到磁带交厂的年代,今天它代表一件事:**设计从"还能改的数据"变成"已经下单、不可撤销的制造订单"。**

> 一句话定位:**流片 = 按下那个几百万美元、不可撤销的按钮——把签核通过的版图交给 foundry,做成掩模,开工。从此设计冻结成硅。**

---

## 1. 流片到底是什么

一句话:**把签核通过的最终版图(GDSII/OASIS),经掩模数据准备做成光掩模,正式提交给晶圆厂开始制造——这是设计与制造的交接点,也是设计冻结、不可逆的节点。**

### 为什么它是一个"重大事件"而非一个"步骤"
- **不可逆 + 高成本**:掩模一旦做出来,改一点就要重做掩模、重新流片(re-spin)——先进节点一套掩模动辄几百万到上千万美元,周期几个月。
- **它是责任的交接**:此前设计团队对正确性负责(靠 ⑨ 签核背书);此后 foundry 对"按版图造出来"负责。
- **它把"数据"变成"物理订单"**:GDSII 不再是可编辑的设计,而是要被刻进石英掩模、印到硅片上的几何。

*在 TPU 上*:⑨ 签核通过的 28nm 版图(GDSII)→ 经掩模数据准备 → 做出整套 28nm 光掩模 → 提交 foundry 开工。15 个月从立项到部署的 TPU v1,流片就是这条时间线上最关键的一个不可逆节点。

---

## 2. 谁来做 + 需要什么输入

**谁来做**:**流片 / 掩模 / 量产接口工程师**对接 **foundry**;由项目以正式的 **tape-out release** 拍板。
**需要什么输入**:
- **签核通过的最终版图(GDSII/OASIS)**(来自 ⑨)+ 各层定义、工艺信息。
- **foundry 的掩模/工艺规范**、流片方式选择(全掩模 / MPW)、商务合同与排期。

> 关键心态:到这一步,正确性已由 ⑨ 背书;流片工程要保证的是**数据正确交付、掩模正确制作、商务与排期落实**——一个交付/格式错误,可能浪费一整轮流片。

---

## 3. 流片的核心内容(尽量穷尽地列)

每项给:**是什么 + 怎么做 + 在 TPU 上长什么样**。

### 3.1 流片与交付数据(GDSII/OASIS)
- **是什么**:把签核版图打包成 foundry 接受的格式(**GDSII / OASIS**)+ 层映射、文档,正式提交。
- **怎么做**:核对层、密度、填充(dummy/fill)、印章/对位标记齐全;一次提交错了代价极大,交付前反复核对。
- *在 TPU 上*:把 28nm die 的完整版图按 foundry 层定义打包提交。

### 3.2 掩模数据准备(MDP:fracturing / OPC / RET)
- **是什么**:版图不能直接做掩模,要经**掩模数据准备**——**fracturing**(转成掩模写设备能写的图形)、**OPC(光学邻近校正)**、**RET(分辨率增强:相移掩模、SRAF、多重曝光)**。
- **怎么做**:因为光的波长画不出那么小的图形,要"预畸变"版图,让印到硅上的结果接近设计;节点越先进,OPC/RET 越复杂。
- *在 TPU 上*:28nm 的图形经 OPC 校正,确保印到硅上的晶体管/连线尺寸准确。

### 3.3 光掩模(mask / reticle)制作
- **是什么**:把处理后的数据**写到掩模**——石英基板上的吸收层(铬等),用电子束/激光写出每一层的图形;**一套掩模 = 每个工艺层一张**(几十张)。注:先进节点用的其实是 **reticle**——图形按 **4× 放大**做在掩模上,光刻时由 scanner 经缩小投影透镜缩到 1× 并逐场(field)步进曝光,所以掩模图形并非与硅 1:1。
- **怎么做**:掩模是制造的"母版",精度要求极高、价格昂贵;做好后要检测/修复缺陷。
- *在 TPU 上*:28nm 工艺的整套掩模(逻辑层 + 多层金属 + 各种工艺层)被制作出来。

### 3.4 流片方式:全掩模 vs MPW / shuttle
- **是什么**:**全掩模(full mask set)**——专属整套掩模,适合量产;**MPW / 多项目晶圆(shuttle)**——多个项目共享一套掩模/晶圆,**摊薄 NRE**,适合原型/小量试产。
- **怎么做**:首版验证/试错常走 MPW 省钱;确认要量产再上全掩模。
- *在 TPU 上*:量产级芯片通常上全掩模;早期试验性硅可借 MPW 降成本(视项目策略)。

### 3.5 NRE 与流片成本
- **是什么**:流片是一次性工程费(NRE)的大头——**掩模成本**在先进节点尤其惊人(可达千万美元级),加上工程、IP、流片服务费。
- **怎么做**:成本直接进商业模型(① 那篇的"盈亏平衡"靠它);所以**一次流片成功(first silicon success)**价值巨大——少一轮 re-spin 省几百万 + 几个月。
- *在 TPU 上*:28nm 一套掩模 + 流片是 TPU 项目 NRE 的重要组成;"15 个月部署"的紧凑时间线,经不起多次 re-spin。

### 3.6 交接、签字与流片后等待
- **是什么**:正式的 **tape-out release**(签字交付)→ foundry 排程开工 → 等待**首批硅(first silicon)**回来(**turnaround time** 通常数周到数月)。
- **怎么做**:交付走 checklist + 签字负责;之后是漫长等待——这期间团队转去做软件、测试程序、bring-up 准备。
- *在 TPU 上*:tape-out 后等 28nm 产线把首批 wafer 跑出来,团队同步准备 bring-up 与软件栈。

---

## 4. 贯穿始终、必须反复权衡的东西

- **不可逆 vs 进度**:晚流片错过市场,早流片带疑点 re-spin——⑨ 签核就是为了让这一步"敢按"。
- **全掩模 vs MPW**:省钱试错 vs 直奔量产,看项目阶段与把握。
- **first silicon success 的价值**:一次成功省下的,是几百万美元和一整代时间窗口。
- **数据交付零容错**:格式/层/填充错一处,可能废掉整轮流片。

> 灵魂句:**流片是设计的"成人礼"——过了这道门,它不再是能随时改的代码和版图,而是即将被刻进硅、几百万美元押上去的物理现实。**

---

## 5. 流片"怎么做"出来的(方法与工具)

- **掩模数据准备(MDP)**:fracturing、OPC、RET 工具(foundry/掩模厂侧)。
- **数据格式**:GDSII / OASIS(版图)、各层映射文件。
- **流程**:签核 release → 数据提交 foundry → MDP → 掩模制作 → 上线;商务/排期合同并行。
- **质量**:交付 checklist、版图最终核对(层/填充/标记)。

---

## 6. 流片阶段的交付物(交给下游什么)

- **提交给 foundry 的最终版图数据**(GDSII/OASIS)+ 层映射/文档。
- **整套光掩模**(由掩模厂/ foundry 制作)。
- **tape-out release 记录 + 商务排期**:制造正式启动的凭证。

> 流片的"产物"是把设计变成 foundry 手里的一套掩模和一张制造订单——下游 ⑪ 制造据此把芯片印到硅上。

---

## 7. 容易被忽视、却致命的坑

1. **带疑点流片**:⑨ 没抠干净就 tape-out → first silicon 出 bug,re-spin 几百万 + 几个月。
2. **数据交付错误**:层映射、填充、对位标记、密度不达标 → foundry 退回或废一轮流片。
3. **低估 OPC/RET 影响**:先进节点不做好分辨率增强,印出来和设计差很多。
4. **掩模缺陷**:掩模本身有缺陷会复制到每片晶圆——掩模检测/修复不能省。
5. **只算掩模成本、忘了周期**:turnaround 几个月,排期没留够会错过窗口。
6. **MPW/全掩模选错**:该试错时直奔全掩模(贵),或该量产了还卡在 MPW(产能不足)。

---

## 8. 小结 + 下一步

**流片 = 把签核通过的最终版图,经掩模数据准备做成光掩模,正式交给晶圆厂开工——设计与制造的交接点,也是设计冻结、几百万美元、不可撤销的节点。** 它的灵魂是 **"过了这道门,设计就冻结成硅,改一点就要 re-spin"**;最该追求的是 **"first silicon success——一次成功省下的是几百万和一整代时间"**;最该零容错的是 **"数据交付与掩模质量"**。

下一步(第 ⑪ 步)是 **制造(Fab)**:晶圆厂用这套掩模,在硅片上经过光刻、沉积、刻蚀、掺杂等数百道工序,**把设计真正"印"成一层层的晶体管和金属互连**——抽象的版图终于变成物理的电路。
