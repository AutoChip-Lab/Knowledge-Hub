# 芯片设计完整流程 · 第 ⑨ 步:签核(Signoff)

> 「芯片设计完整流程」学习系列第九篇。
> 编号主流程:① 产品/市场需求 → ② 系统规格 → ③ 架构 → ④ 微架构 → ⑤ RTL → ⑥ 验证
> → ⑦ 综合 → ⑧ 物理设计 → ⑨ 签核 → ⑩ 流片 → ⑪ 制造 → ⑫ 封测 → ⑬ bring-up/量产。
> DFT设计涉及:③ 测试架构与覆盖率目标 → ④/⑤ 测试控制和可测逻辑 → ⑦ Scan/压缩/MBIST 插入与 ATPG → ⑧ Scan 链重排及测试时钟、功耗、拥塞收敛 → ⑨ 测试模式与覆盖率签核 → ⑫/⑬ ATE、诊断与良率学习。

---

## 0. 先看全景:芯片从想法到出片,大致这几步

```
① 产品/市场需求 → ② 系统规格(Spec) → ③ 架构设计 → ④ 微架构设计 → ⑤ RTL 设计
   → ⑥ 功能验证 → ⑦ 逻辑综合 → ⑧ 物理设计(布局规划/布局/时钟树/布线)
   → ⑨ 签核(STA/功耗/EM-IR/物理验证/等价性) → ⑩ 流片(tape-out)
   → ⑪ 制造(Fab) → ⑫ 封装测试 → ⑬ 芯片回来后 bring-up/量产

DFT设计:③ 测试架构 → ④/⑤ 测试控制和可测逻辑 → ⑦ Scan/压缩/MBIST 与 ATPG → ⑧ Scan 链重排/测试时序功耗 → ⑨ 测试模式与覆盖率签核 → ⑫/⑬ ATE/诊断/良率学习
```

**签核夹在「物理设计(⑧)」和「流片(⑩)」之间**:后端给出了一份"看起来收敛"的版图,但能不能真的造、真的在所有条件下都工作?签核就是流片前**最后、最严格、用"金标准"工具的全面体检**——把时序、功耗、电源完整性、信号完整性、物理规则、与 RTL 的等价性,在最坏条件下逐项确认到全绿。**全绿,才敢按下流片那个不可撤销的按钮。**

> 一句话定位:**前面每一步的"差不多",到签核必须变成"确定";签核不接受"应该没问题"——因为它把守的是流片这道不可逆、几百万美元的门。**

---

## 1. 签核到底是什么

一句话:**用签核级(sign-off quality)的工具,在最严格的条件(多角多模、最坏 corner、片上偏差)下,对版图做全维度的最终验证——时序/功耗/电源/信号/物理/功能等价性全部干净,才放行流片。**

### 为什么这一步不可省、不可松
- **流片不可逆**:掩模一旦做出来,bug 就刻进硅里了,救不回——只能重新流片(re-spin),代价是几百万美元 + 几个月。
- **签核 = 真账**:前面综合/后端用的时序是逐步逼近真实;签核用**真实寄生(SPEF)+ 全 corner + 片上偏差**,这才是最终裁定。**签核说过,才是真的过。**
- **"金标准"工具**:签核用的是被 foundry/业界认可的签核级工具(如 STA 用 PrimeTime),结果具有"可签字负责"的权威性。

### 签核不是单项,而是"全维度同时干净"
签核要同时通过一整组检查(下面 §3 逐项),**任何一项红,就不能流片**。它是一道汇总闸门:把所有维度的"确定无误"集齐,才出具 tape-out 放行。

*在 TPU 上*:⑧ 给出 28nm 收敛版图;⑨ 要用真实寄生在所有 28nm corner 上证明 700MHz 时序成立、阵列满负荷下 IR-drop 不超标、功耗达标、DRC/LVS 全清、门级网表仍 ≡ 已验证的 RTL——全绿,才 tape-out。

---

## 2. 谁来做 + 需要什么输入

**谁来做**:**签核 / STA / 物理验证团队**,联合后端、功耗、可靠性团队;最终由项目以一个正式的 **tape-out 评审**拍板。

**需要什么输入**:
- **最终版图(⑧)**:GDSII + 最终门级网表。
- **真实寄生(SPEF)**:从版图提取的精确 RC。
- **多角多模库与约束**:各 PVT corner 的时序库、SDC、功能/scan shift/at-speed capture/MBIST 等工作模式定义。
- **DFT 基线**:最终 scan chain/ScanDEF、OCC/测试时钟、MBIST/JTAG 模式、DFT DRC/连接/覆盖率报告与最终 ATPG 图样版本。
- **功耗/电源数据**、**工艺的 DRC/LVS 规则与 DFM 要求**、**RTL(做等价性对照)**。

> 关键心态:签核是"宁可多疑"的角色——它默认设计有问题,直到所有维度都证明没问题。它不优化设计,只**裁定能不能流片**。

---

## 3. 签核的核心内容(尽量穷尽地列)

每项给:**是什么 + 怎么做(经验法则) + 在 TPU 上长什么样**。

### 3.1 签核流程与"金标准"工具
- **是什么**:整套签核的组织——准备真实寄生与多角多模数据 → 并行跑各维度签核 → 汇总到一道 tape-out 闸门;用业界认可的签核级工具。
- **怎么做**:每个维度都跑到"零违例"或"违例都被解释/豁免";签核高度迭代——发现问题就回 ⑧ 做 ECO 再签,直到全绿。
- *在 TPU 上*:对 28nm 版图准备 SPEF + 全 corner,组织时序/功耗/电源/物理/等价各路签核。

### 3.2 静态时序分析(STA)——签核主战场
- **是什么**:**STA(Static Timing Analysis)**——不靠仿真,用真实寄生**穷举分析所有时序路径**,逐条检查 **setup(够不够快)和 hold(会不会太快)**。
- **怎么做**:STA 是 vectorless 的(不需要激励),一次覆盖所有时序路径,**就"路径覆盖"而言比仿真彻底**;但它**不懂功能**——会把逻辑上永不敏化的 **false path**、功能上允许多拍的 **multicycle path** 也按单拍真路径分析,必须用 **SDC 时序例外**正确排除(标错会过度悲观、或漏判真违例)。功能模式之外,还必须签 scan shift、scan enable、OCC launch/capture、MBIST/JTAG 等测试模式与跨域 lockup 路径。**setup 失败可降频缓解,hold 失败是硬伤(硅上随机错)**,必须全清。
- *在 TPU 上*:用真实寄生证明阵列与全局路径在 700MHz 下 setup/hold 全部满足——这是 700MHz 这个数字"作数"的最终依据。

### 3.3 多角多模与片上偏差(MMMC / OCV)
- **是什么**:**MMMC(Multi-Mode Multi-Corner)**——在多个**工作模式**(功能/测试/低功耗…)× 多个 **PVT corner**(工艺 SS/TT/FF、电压、温度)下都签核;**OCV/AOCV/POCV**——考虑同一 die 内的**片上偏差**并加 derate。
- **怎么做**:必须在**最坏组合**下仍成立(慢工艺低压高温查 setup、快工艺查 hold);corner 漏一个,硅上就可能在那个条件下失效。
- *在 TPU 上*:28nm 的各 corner × 各模式都要过——数据中心高温、电压波动下 700MHz 仍稳。

### 3.4 功耗签核(动态 + 漏电)
- **是什么**:确认芯片功耗在预算内——**动态功耗(翻转)+ 静态功耗(漏电)**,看**峰值与平均**;可基于活动向量或 vectorless。
- **怎么做**:核对是否超 TDP、是否触发散热/供电上限;峰值功耗还关系到电源完整性(下条)。测试模式也要独立分析——scan shift/capture 的大面积同时翻转可能比典型功能负载更激进,必须与 ATPG 的低功耗约束一致。
- *在 TPU 上*:确认整芯片满负荷功耗约 40W(忙时;空闲约 28W)、远在 75W 的 TDP 预算之内、漏电受控(多 Vt 的效果在这里验收)。

### 3.5 电源完整性(IR-drop / EM)
- **是什么**:**电源完整性签核**——**IR-drop**(电流流过电源网格造成的压降,静态+动态)与**电迁移(EM)**(电流密度过高导致金属线长期失效)。
- **怎么做**:IR-drop 过大 → 单元实际电压不足 → 时序失效甚至功能错;EM 超标 → 影响芯片寿命。高电流块(算力阵列)是重灾区,要在签核确认 PDN 扛得住。
- *在 TPU 上*:256×256 阵列满负荷瞬时电流巨大,IR-drop/EM 签核确认 ⑧ 铺的电源网格真的压得住、用得久。

### 3.6 信号完整性(SI)签核
- **是什么**:**SI 签核**——**串扰(crosstalk)**引起的延迟变化和噪声毛刺(glitch)是否会破坏时序或功能。
- **怎么做**:把串扰效应纳入 STA(crosstalk-aware);先进节点 SI 越来越主导,必须专门签核。
- *在 TPU 上*:阵列里密集并行的连线之间串扰、长总线噪声都要确认不破坏 700MHz 时序。

### 3.7 物理验证签核(DRC / LVS / antenna)
- **是什么**:版图的最终硬性体检——**DRC**(设计规则)、**LVS**(版图 ≡ 网表)、**antenna**、ERC 等,用签核级工具做**最终金标准**确认。
- **怎么做**:**必须 100% 干净**(违例要么修掉、要么有 foundry 认可的豁免)——这是 foundry 收版图、能否流片的硬门槛。
- *在 TPU 上*:整块 28nm die 的 DRC/LVS/antenna 全清,版图才被 28nm 产线接收。

### 3.8 逻辑等价性签核(LEC)
- **是什么**:**最终等价检查**——证明经过综合、后端、所有 ECO 之后的**最终门级网表,仍和已验证的 RTL 逻辑等价**。
- **怎么做**:后端的每次物理优化/ECO 都可能动逻辑,**只有 LEC 能保证功能没被悄悄改坏**;签核必须有这一项兜底,把功能正确性从 ⑥ 一路守到 tape-out。
- *在 TPU 上*:证明最终版图对应的网表 ≡ 当年 ⑥ 验过的 RTL——功能正确性完整守到流片。

### 3.9 可靠性与可制造性(DFM / ESD / latch-up / aging)
- **是什么**:面向**良率与长期可靠**的签核——**DFM(可制造性)**优化(冗余 via、光刻友好)、**ESD/latch-up** 保护检查、**老化(aging:HCI/NBTI)**等可靠性裕量。
- **怎么做**:DFM 提升良率、降单位成本;ESD/latch-up/aging 关系到芯片能不能在现场长期稳定工作(尤其数据中心 7×24)。
- *在 TPU 上*:数据中心要长期高可靠 → 这些可靠性/良率签核确保 TPU 量产良率与寿命达标。

### 3.10 签核闸门与 tape-out 决策
- **是什么**:把所有维度的签核结果**汇总成一道正式闸门**——包括功能模式和 DFT 测试模式;全绿(或违例都被解释/豁免)才出具 **tape-out 放行**。
- **怎么做**:这是一个**正式评审**,有 checklist、有签字负责;除常规 STA/功耗/物理验证/LEC 外,还核对最终 scan/MBIST/JTAG 连通性、test-mode STA/IR-EM、chain/ECO 与 ATPG 图样版本一致性,再由签核团队、DFT、后端和项目共同确认"可以流片"。
- *在 TPU 上*:所有签核绿灯 → tape-out 评审通过 → 把 28nm GDSII 送 foundry。

---

## 4. 贯穿始终、必须反复权衡的东西

- **严格 vs 进度**:签核越严越安全,但每轮签核+ECO 都耗时;漏签一项又可能整代翻车。把握"足够严"的度。
- **最坏 corner 思维**:不是"典型情况能跑",而是"所有 corner、所有模式、片上偏差下都成立"——签核按最坏情况裁定。
- **setup 可救 vs hold 是硬伤**:setup 违例能靠降频缓解,hold 违例是不可降频救的硅上随机错,必须零容忍。
- **违例 vs 豁免**:不是所有违例都要修,有些可由专家解释/foundry 豁免;但豁免要有据、可签字。
- **不可逆的敬畏**:tape-out 不可撤销——签核宁可多疑、多跑一轮,也别带疑点流片。

> 灵魂句:**签核是按下不可逆按钮前的最后一双眼睛——它默认设计有问题,直到每一个维度、每一个最坏 corner 都证明没问题;它不让设计更好,只决定它配不配流片。**

---

## 5. 签核"怎么做"出来的(方法与工具)

- **时序签核(STA)**:PrimeTime(Synopsys)、Tempus(Cadence)——多角多模、含真实寄生与 OCV。
- **功耗签核**:PrimePower、Voltus。
- **电源完整性(IR/EM)**:RedHawk、Voltus —— rail 分析。
- **物理验证**:Calibre、IC Validator、Pegasus —— DRC/LVS/antenna/DFM。
- **等价性(LEC)**:Formality、Conformal。
- **寄生提取**:StarRC、Quantus(出 SPEF)。
- **流程**:并行跑各维度 → 汇总违例 → 回 ⑧ ECO → 再签,迭代到全绿 + tape-out 评审。

---

## 6. 签核阶段的交付物(交给下游什么)

- **各维度签核报告**(STA/功耗/IR-EM/SI/物理验证/LEC/可靠性),全部干净或违例已豁免。
- **tape-out 签核 checklist + 放行决议**:正式的"可以流片"凭证。
- **最终 GDSII + 相关数据**:交给 **⑩ 流片**送 foundry。

> 签核的产物不是一个文件,而是**一份"敢流片"的、可签字负责的确定性**——它把整条流程所有维度的正确性,集中确认到可以承担那个不可逆决定。

---

## 7. 容易被忽视、却致命的坑

1. **漏一个 corner / 模式**:典型情况过了,却在某个 PVT 角或某个模式下失效——MMMC 不全是签核头号坑。
2. **hold 违例没清干净**:hold 不能靠降频救,漏掉就是硅上随机错,芯片报废。
3. **IR-drop / EM 低估**:高电流块压降致时序/功能失效,或 EM 缩短寿命——电源完整性最易被轻视。
4. **SI / 串扰没纳入签核**:真实噪声让时序在硅上失效,纯静态时序看不到。
5. **不做最终 LEC**:后端 ECO 悄悄改了功能,带 bug 流片。
6. **DRC/LVS 留尾巴**:foundry 直接拒收,或带隐患流片。
7. **把签核当形式**:走过场、不较真最坏情况 → 流片后才暴露,re-spin 代价巨大。
8. **豁免无据**:把违例随意豁免却没有可签字的依据 → 风险被掩盖。
9. **签核与后端脱节**:违例反复在 ⑧⑨ 之间打转、不收敛,拖垮进度。

---

## 8. 小结 + 下一步

**签核 = 流片前最后、最严格的全维度体检——用金标准工具,在多角多模、最坏 corner、片上偏差下,确认时序(STA)、功耗、电源完整性(IR/EM)、信号完整性、物理规则(DRC/LVS)、与 RTL 的等价性全部干净,出具可签字负责的 tape-out 放行。** 它的灵魂是 **"签核不接受'应该没问题',只接受'每个最坏 corner 都证明没问题'"**;最该敬畏的是 **"tape-out 不可逆,宁可多疑"**;最该零容忍的是 **"hold 违例和 DRC/LVS 不干净"**。

下一步(第 ⑩ 步)是 **流片(tape-out)**:把这份签核通过的最终版图(GDSII)正式交付给晶圆厂(foundry)——制作光掩模(mask),设计就此从"数据"变成即将开工的"制造订单",几百万美元、不可撤销。
